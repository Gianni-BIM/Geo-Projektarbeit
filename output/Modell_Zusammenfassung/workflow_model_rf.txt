======================================================================
WORKFLOW: RANDOM FOREST MODELL ZUR ERKLÄRUNG DES SHI
======================================================================

1. ZIEL DES MODELLS
----------------------------------------------------------------------

Ziel ist es, den Soil Health Index (SHI) als Funktion mehrerer Umwelt-
faktoren zu modellieren und zu erklären:

SHI = f(Klima, Landnutzung, Topographie, Geologie)

Das Modell soll:
- SHI vorhersagen (Prediction)
- Einflussfaktoren identifizieren (Feature Importance)
- Interaktionen erkennen (Data Mining)


2. DATENBASIS
----------------------------------------------------------------------

Input-Daten:
- SHI (Zielvariable)
- Temperatur
- Niederschlag
- Höhe
- Landnutzung (factor)
- Landbedeckung (factor)
- Klima (Köppen-Geiger)

Jede Zeile = ein Standort


3. DATENVERARBEITUNG
----------------------------------------------------------------------

Schritte:

1. Entfernen von:
   - Koordinaten (X/Y)
   - IDs

2. Filter:
   - Kategorien mit <30 Beobachtungen entfernen

3. Transformation:
   - Faktoren als categorical (kein One-Hot-Encoding notwendig)

4. Final:
   - X = alle Variablen
   - y = SHI


4. EXPLORATIVE DATENANALYSE (EDA)
----------------------------------------------------------------------

- Korrelationsmatrix → prüft Abhängigkeiten
- Boxplots → Unterschiede zwischen Gruppen
- Histogramm → Verteilung des SHI

Ziel:
→ prüfen, ob Daten sinnvoll modellierbar sind


5. MODELL (RANDOM FOREST / CFOREST)
----------------------------------------------------------------------

Das Modell basiert auf:

- Bootstrapping:
  zufällige Stichproben (fraction = 0.632)

- Entscheidungsbäume:
  rekursive Aufteilung der Daten

- Splits:
  Auswahl der Variable mit maximaler Varianzreduktion

WICHTIG:
→ Ziel ist NICHT optimale Böden zu finden
→ sondern Unterschiede im SHI zu erklären


6. SPLIT-LOGIK (HOBLEY)
----------------------------------------------------------------------

Jeder Split erfüllt:

- Minimierung der Varianz innerhalb der Gruppen
- Maximierung der Unterschiede zwischen Gruppen

Beispiel:

Rain < 800 → Gruppe A
Rain > 800 → Gruppe B

→ beide Gruppen homogener als vorher


7. CONDITIONAL INFERENCE
----------------------------------------------------------------------

Splits erfolgen nur wenn:

p < 0.001 (approx. über mincriterion)

Vorteil:
- verhindert Bias
- reduziert Overfitting


8. HYPERPARAMETER
----------------------------------------------------------------------

Wichtige Parameter:

- ntree        → Anzahl der Bäume
- mtry         → Variablen pro Split
- mincriterion → Signifikanzniveau
- fraction     → Stichprobengröße

Optimierung:
→ Grid Search + OOB R²


9. BOOTSTRAPPING
----------------------------------------------------------------------

Jeder Baum nutzt:

zufällige Stichprobe des Datensatzes

→ erzeugt unterschiedliche Verteilungen

→ erhöht Robustheit


10. MODELLBEWERTUNG
----------------------------------------------------------------------

- OOB R²:
  erklärte Varianz

- OOB RMSE:
  Vorhersagefehler

- Observed vs Predicted:
  Modellgüte


11. VARIANZ-ERKLÄRUNG
----------------------------------------------------------------------

R² = 1 - (MSE / Varianz)

→ zeigt Anteil erklärter Varianz

→ typisch bei Geo-Daten: 0.3 – 0.6


12. FEATURE IMPORTANCE
----------------------------------------------------------------------

Bestimmung der wichtigsten Faktoren:

- basiert auf Permutation
- misst Einfluss auf Modellgüte

Ergebnis:
→ Ranking der Einflussfaktoren


13. INTERAKTIONEN
----------------------------------------------------------------------

Das Modell erkennt automatisch:

- Klima × Landnutzung
- Niederschlag × Temperatur
- Höhe × Klima

→ wichtig für Forschungsfrage


14. DATA MINING
----------------------------------------------------------------------

Nach Training:

- wichtigste Faktoren analysieren
- Kombinationen untersuchen
- Szenarien ableiten

Beispiel:
hoher SHI → feucht + Wald


15. MODELLLOGIK (WICHTIG!)
----------------------------------------------------------------------

Der Random Forest:

❌ sucht NICHT optimale Bedingungen
✅ erklärt Unterschiede im SHI

→ zentrale Hobley-Anforderung erfüllt


16. WORKFLOW ZUSAMMENGEFASST
----------------------------------------------------------------------

Daten → Cleaning → EDA → RF Training → Optimierung →
Evaluation → Interpretation → Data Mining
